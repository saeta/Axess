Success Stories A few victories.

Non-Javascript Login for CourseWare 3/19/12

When I went to write the models.CourseWareSite login module, Selenium's HtmlUnit driver didn't run Javascript. Because the normal code path for Javascript-enabled browsers didn't work, I turned off Javascript in order to debug. It turns out, the non-javascript login had been neglected, and instead routed people to federation.org, a domain name unaffiliated, and outside of anyone's control.

When I emailed the CourseWare people, they were very fast on the draw, and fixed the issue within a couple hours. Good work!

Axess passes Axess 3/19/12

I demo'd this tool to my girlfriend, and bragged about how it finds all these easy-to-fix accessibility errors. She turned around and challenged me to run it on itself. So I did. And found a few bugs of my own! As of this writing, Axess comes back clean.
